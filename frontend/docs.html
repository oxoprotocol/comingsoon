OXO Protocol

OXO is a Layer-1.5 protocol that brings Bitcoin into the EVM ecosystem without centralized custodians. Users bridge BTC to receive oxoBTC — a fully collateralized wrapped token — which can be swapped, staked, used as lending collateral, or exited instantly back to BTC via Flash Exit. ZK rollups provide scalable L2 transfers at minimal cost.

Core Primitives

  • oxoBTC — 1:1 BTC-backed ERC-20 (8 decimals), minted by the Bridge relayer upon confirmed BTC deposit
  • OXO — governance & rewards token (50M supply, 18 decimals)
  • Bridge — ECDSA-signed mint/burn system for BTC ↔ ETH transfers
  • Swap — UniswapV2 fork (Factory + Router) with on-chain slippage protection
  • Staking — dual-pool yield for oxoBTC stakers (earn OXO) and OXO stakers (earn ETH + oxoBTC)
  • Lending — collateralized borrowing: deposit oxoBTC, borrow ETH (70% LTV)
  • Flash Exit — single-tx or chunked ETH→oxoBTC→BTC exit path
  • ZK Rollup — Groth16-verified L2 state root for low-cost transfers
Testnet Notice: OXO Protocol is currently live on Sepolia Testnet. All contracts are verified and all tests pass. Mainnet launch will follow a full audit cycle.

Architecture

OXO runs as a coordinated stack across three layers:

  • Bitcoin Layer — user deposits BTC to a protocol address; mempool.space WebSocket monitors confirmations
  • L1 (Sepolia) — smart contracts handle minting, swapping, staking, lending, and ZK proof verification
  • L2 (Rollup) — off-chain sequencer batches transfers and submits state roots to OXORollup.sol

Relayer

A Node.js backend (backend/server.js) acts as the permissioned relayer. It watches BTC deposits via mempool.space WebSocket and Blockstream API, then calls mintAndTransfer() on the Bridge with an ECDSA signature. The relayer also listens for RedeemRequested events to initiate BTC sends on the Bitcoin network.

Fee Flow

A 0.1% bridge fee is routed to ProtocolTreasury.sol, which distributes incoming oxoBTC fees as: 70% to stakers (Pool 1), 20% to Protocol-Owned Liquidity, and 10% to a reserve.

Bridge

The bridge enables trustless-verified BTC ↔ ETH transfers using ECDSA cryptography. No oracles or multisig wallets are required for mint authorization — the relayer's signature is verified on-chain.

BTC → oxoBTC (Deposit)

  • Register your ETH address via POST /login with your BTC address
  • Send BTC to the protocol's deposit address (shown as QR in the app)
  • The relayer detects the deposit after 1 confirmation
  • Relayer calls mintAndTransfer(to, amount, depositId, signature)
  • oxoBTC is minted to your wallet; digest is marked used (replay protection)

oxoBTC → BTC (Redeem)

  • Approve the Bridge contract to spend your oxoBTC
  • Call withdraw(btcAddress, amount) — oxoBTC is burned immediately
  • A RedeemRequested event is emitted; relayer picks it up
  • Relayer sends BTC on the Bitcoin network and calls markRedeemCompleted()
ECDSA digest: keccak256(abi.encode(chainId, bridgeAddress, to, amount, depositId)) — signed with EIP-191 personal_sign prefix.

Swap & Pools

OXO uses a UniswapV2 fork (Factory + Router) deployed on Sepolia. Two primary pools are seeded at launch: oxoBTC/WETH and OXO/WETH.

Swap

  • Select input/output tokens, enter amount
  • Price impact and minimum output shown pre-confirm (50 bps slippage default)
  • Calls swapExactTokensForTokens() or swapExactETHForTokens()

Add Liquidity

  • Deposit oxoBTC + ETH in the current pool ratio
  • LP token preview shown before confirming (300 bps slippage for adds)
  • Call addLiquidity() or addLiquidityETH() on Router

Remove Liquidity

  • Enter LP token amount or percentage
  • Expected oxoBTC + ETH output shown
  • Call removeLiquidity() on Router

Workflow

  • Deposit — approve + call depositCollateral(amount)
  • Borrow — call borrow(ethAmount) — ETH sent to your wallet
  • Repay — send ETH + call repay() — interest is included
  • Withdraw — call withdrawCollateral(amount) — only if LTV stays safe
  • Liquidate — any user can liquidate undercollateralized positions by repaying debt and receiving collateral + bonus

Flash Exit

Flash Exit provides a single-transaction path from ETH to BTC. It wraps a swap (ETH→oxoBTC via the Router) and a bridge redeem in one call.

Instant Exit

For smaller amounts — swaps all ETH to oxoBTC, then immediately calls withdraw() on the Bridge. The entire sequence happens atomically in one tx.

Chunked Exit

For large amounts where a single swap would cause excessive slippage — exitChunked() uses binary search to find the optimal chunk size and executes swaps at 30-second intervals.

Get Quote

Call getExitQuote(ethAmount) to see: number of chunks, estimated oxoBTC output, and slippage bps — before committing.

ZK Rollup

OXO's L2 uses a Groth16-based ZK rollup for high-throughput, low-cost oxoBTC transfers between registered accounts.

Circuit

  • 4 transactions per batch, 8 accounts in the Merkle tree
  • State root: Poseidon(balances[0..7]) computed via circomlibjs
  • Proof system: Groth16 (snarkjs + circom v2.2.3)

Workflow

  • L1 Deposit — send oxoBTC to OXORollup.sol; sequencer credits L2 balance
  • L2 Transfer — submit transfer to sequencer; batched into next proof
  • L2 Withdraw — request withdrawal; sequencer includes in batch; oxoBTC released on L1 after proof verification
Note: The rollup sequencer currently runs locally (localhost:3002). A public sequencer endpoint will be available with the mainnet launch.

Security

Replay Protection

Every bridge deposit digest is stored in usedDigests mapping. Re-submitting the same deposit ID reverts with BRIDGE: TXID_ALREADY_USED.

Role-Based Access

  • DEFAULT_ADMIN_ROLE — deployer; manages all roles and parameters
  • SIGNER_ROLE — backend relayer; can mint oxoBTC and complete redeems
  • MINTER_ROLE (oxoBTC) — Bridge contract only

Pausable

Bridge, Lending, and Staking contracts inherit OpenZeppelin Pausable. Admin can pause all user-facing operations in an emergency.

Audits

Testnet only. OXO Protocol has not yet undergone a formal security audit. Do not use with real funds. A third-party audit is planned before mainnet deployment.
